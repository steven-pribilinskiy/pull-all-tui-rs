<p class="term-rule not-content">what it does</p>

<ul class="feat not-content">
  <li>
    <span class="feat__glyph feat__glyph--ok" aria-hidden="true">✓</span>
    <h3>Pull a fleet at once</h3>
    <p>Every repo fast-forwards in parallel, bounded by a concurrency limit (default: <code>nproc</code>), with live per-repo log streaming. Auto-pull on launch is configurable: it pulls small flat sets and skips big trees or the directory-tree view.</p>
  </li>
  <li>
    <span class="feat__glyph feat__glyph--cyan" aria-hidden="true">◌</span>
    <h3>Useful before it pulls</h3>
    <p>A persisted status cache seeds the list the instant repos are discovered: last-known status shown dim with an age (<code>up-to-date 2d</code>) until you pull. No network needed to know where everything stands.</p>
  </li>
  <li>
    <span class="feat__glyph feat__glyph--magenta" aria-hidden="true">⑂</span>
    <h3>Manage each repo</h3>
    <p>A full-screen page per repo lists every branch, worktree, and stash with fresh ahead/behind. Check out, fast-forward, delete, or discard (with confirms), and read inline diffs in a 90%-of-screen modal.</p>
  </li>
  <li>
    <span class="feat__glyph feat__glyph--warn" aria-hidden="true">⌨</span>
    <h3>Keyboard-first, mouse-aware</h3>
    <p>Vim-style navigation, a searchable keymap, sortable columns, name and status filters, and clickable status-bar commands. Jump into <a href="https://github.com/jesseduffield/lazygit">lazygit</a> on any repo with <kbd>l</kbd>. Runs headless with <code>--no-tui</code>.</p>
  </li>
</ul>

<p class="term-rule not-content">quick start</p>

```bash
# Install (Linux/macOS; on Windows use WSL)
curl -fsSL https://steven-pribilinskiy.github.io/polygit/install.sh | bash

# Run it in any tree of repos
polygit ~/projects
```
