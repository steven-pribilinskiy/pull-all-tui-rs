## Notes

- **Retry vs refetch** — `r`/`R` only re-run repos that need it (failed or skipped);
  `e`/`E` re-run any repo even if it was already up to date. In the status bar, `r`/`R`
  dim when no repo has an issue, and `e`/`E` dim when nothing is eligible.
- **Sort the list** — press `s` then a column key (or click a column header) to sort by
  name / status / ahead-behind / dirty / last-commit / worktrees / branches / stashes;
  re-pick or re-click flips `▲`/`▼`. The order persists across runs.
- **Status filter moved to `f`** — `f` then `a`/`u`/`c`/`s`/`f`/`i` filters the list by
  status (on top of the `/` name filter); `s` is now the sort leader.
- **Panes** — the focused pane (`Tab`, or `1`/`2`) shows a bright rounded border and a `[1]`/`[2]`
  label; the divider shows a grip and highlights while you drag it.
- **`l` opens lazygit** — suspends the TUI and runs lazygit in the selected repo (or the selected
  row's path on the repo page), returning when you quit it. If lazygit isn't installed, a note
  appears instead.
- **Checkout moved to `Shift`+`Enter`** — on the repo page, plain `Enter` (and a
  double-click) opens the diff for a stash or dirty row. `Shift`+`Enter` checks out the
  selected clean, non-current branch, so the two actions no longer collide.

  :::caution[Terminal support]
  `Shift`+`Enter` relies on the Kitty keyboard protocol to be reported distinctly.
  `pull-all` enables it when the terminal supports it (recent Windows Terminal, WezTerm,
  kitty, foot). On terminals without support, `Shift`+`Enter` falls back to plain `Enter`.
  :::
- **Clickable commands** — the action hints in the status bar (and the `t` column menu)
  are mouse-clickable; clicking runs the same command as the key.
- **Scrollbars** — the repo list, the preview, the help modal, and the repo page all
  show a scrollbar when their content overflows.
